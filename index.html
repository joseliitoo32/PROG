1. Clases y Objetos

Las clases son **plantillas** para crear objetos y permiten definir atributos y comportamientos comunes. Los objetos son instancias concretas de estas clases.

  • Las clases permiten reutilizar código.
  • Los objetos almacenan datos específicos.
  • Facilitan la programación modular y orientada a objetos.

// Clase Persona
public class Persona {
    private String nombre;
    private int edad;

    // Constructor por defecto
    public Persona() {
        this.nombre = "Anónimo";
        this.edad = 18;
    }

    // Constructor con parámetros
    public Persona(String nombre, int edad) {
        this.nombre = nombre;
        this.edad = edad;
    }

    // Getters y setters
    public String getNombre() { return nombre; }
    public void setNombre(String nombre) { this.nombre = nombre; }
    public int getEdad() { return edad; }
    public void setEdad(int edad) { if (edad>=0) this.edad = edad; }

    @Override
    public String toString() {
        return "Hola, soy " + nombre + " y tengo " + edad + " años.";
    }
}

// Uso de la clase
Persona p1 = new Persona("Jose", 22);
System.out.println(p1);

La encapsulación protege los atributos de la modificación directa. Los getters y setters permiten controlar y validar los valores.

2. Constructores y Sobrecarga

Los constructores inicializan los objetos. La sobrecarga permite tener varios constructores con distintos parámetros.


public class Libro {
    private String titulo;
    private String autor;
    private int paginas;

    // Constructor básico
    public Libro() {
        this.titulo = "Desconocido";
        this.autor = "Desconocido";
        this.paginas = 0;
    }

    // Constructor sobrecargado
    public Libro(String titulo, String autor) {
        this.titulo = titulo;
        this.autor = autor;
        this.paginas = 100; // valor por defecto
    }

    // Constructor completo
    public Libro(String titulo, String autor, int paginas) {
        this.titulo = titulo;
        this.autor = autor;
        this.paginas = paginas;
    }
}

// Uso
Libro l1 = new Libro();
Libro l2 = new Libro("1984", "George Orwell");
Libro l3 = new Libro("El Quijote", "Cervantes", 863);

La sobrecarga de constructores permite flexibilidad en la creación de objetos, según la información disponible.

3. Encapsulación

Consiste en proteger los atributos usando private y controlar el acceso mediante métodos públicos.


class CuentaBancaria {
    private double saldo;

    public CuentaBancaria(double saldoInicial) {
        this.saldo = saldoInicial;
    }

    public double getSaldo() { return saldo; }

    public void depositar(double cantidad) {
        if(cantidad>0) saldo += cantidad;
    }

    public void retirar(double cantidad) {
        if(cantidad>0 && cantidad <= saldo) saldo -= cantidad;
    }
}

// Uso
CuentaBancaria c = new CuentaBancaria(500);
c.depositar(200);
c.retirar(100);
System.out.println(c.getSaldo()); // 600

La encapsulación asegura que los cambios en los atributos ocurran solo a través de métodos controlados, evitando errores y manteniendo integridad de datos.

4. Enumerados (Enums)

Los enumerados definen un conjunto limitado de valores posibles. Son útiles para opciones fijas como días, géneros, estados, etc.


// Enumerado Día de la semana
public enum Dia { LUNES, MARTES, MIERCOLES, JUEVES, VIERNES, SABADO, DOMINGO; }

// Uso en clase
class Evento {
    private String nombre;
    private Dia dia;

    public Evento(String nombre, Dia dia) {
        this.nombre = nombre;
        this.dia = dia;
    }
}

// Ejemplo
Evento e1 = new Evento("Clase de Java", Dia.LUNES);
Evento e2 = new Evento("Concierto", Dia.VIERNES);

Los enums evitan asignaciones inválidas y hacen el código más legible y seguro.

5. Métodos y Atributos Estáticos

Los elementos estáticos pertenecen a la clase en lugar de a los objetos individuales. Se usan para constantes, contadores o utilidades.


// Contador de objetos
class Contador {
    private static int total = 0;

    public Contador() { total++; }

    public static void mostrarTotal() {
        System.out.println("Objetos creados: " + total);
    }
}

// Uso
new Contador();
new Contador();
Contador.mostrarTotal(); // Objetos creados: 2

Los métodos y atributos estáticos permiten acceder a información común sin instanciar la clase.

6. Herencia

La herencia permite crear clases que reutilizan atributos y métodos de otra clase.


// Clase base
class Vehiculo {
    protected String marca;
    public Vehiculo(String marca) { this.marca = marca; }
    public void mostrarMarca() { System.out.println("Marca: " + marca); }
}

// Clase derivada
class Coche extends Vehiculo {
    private int puertas;
    public Coche(String marca, int puertas) { super(marca); this.puertas = puertas; }
    public void mostrarPuertas() { System.out.println("Puertas: " + puertas); }
}

// Uso
Coche c = new Coche("Toyota", 4);
c.mostrarMarca();
c.mostrarPuertas();

La herencia reduce duplicación de código y permite especializar clases según necesidades.

7. Polimorfismo

El polimorfismo permite que un objeto se comporte de distintas formas según su tipo real en tiempo de ejecución.


abstract class Animal {
    public abstract void sonido();
}

class Perro extends Animal {
    @Override
    public void sonido() { System.out.println("Guau"); }
}

class Gato extends Animal {
    @Override
    public void sonido() { System.out.println("Miau"); }
}

// Uso polimórfico
Animal a1 = new Perro();
Animal a2 = new Gato();
a1.sonido(); // Guau
a2.sonido(); // Miau

El polimorfismo permite escribir código más flexible y extensible.

8. Composición

La composición indica que una clase contiene objetos de otras clases, formando relaciones de tipo “tiene un”.


class Motor {
    private int caballos;
    public Motor(int caballos) { this.caballos = caballos; }
    public int getCaballos() { return caballos; }
}

class Coche {
    private String modelo;
    private Motor motor;
    public Coche(String modelo, Motor motor) { this.modelo = modelo; this.motor = motor; }
    public void mostrarInfo() {
        System.out.println("Modelo: " + modelo + ", Motor: " + motor.getCaballos() + " CV");
    }
}

// Uso
Motor m = new Motor(120);
Coche c = new Coche("Honda", m);
c.mostrarInfo();

La composición permite reutilizar clases y crear relaciones claras entre objetos.

9. Interfaces

Las interfaces definen un contrato que las clases deben cumplir, sin implementar lógica. Permiten polimorfismo y desacoplamiento.


interface Volador {
    void volar();
}

class Pajaro implements Volador {
    @Override
    public void volar() { System.out.println("El pájaro vuela"); }
}

class Avion implements Volador {
    @Override
    public void volar() { System.out.println("El avión vuela"); }
}

// Uso
Volador v1 = new Pajaro();
Volador v2 = new Avion();
v1.volar(); // El pájaro vuela
v2.volar(); // El avión vuela

Las interfaces permiten que distintos tipos de objetos compartan métodos comunes, mejorando flexibilidad y mantenimiento.

10. Final, equals(), hashCode() y toString()

Estos elementos son esenciales para control de clases, comparación de objetos y representación textual.


final class Constantes {
    public static final double PI = 3.1416;
}

class Persona {
    private String nombre;
    private int edad;

    @Override
    public boolean equals(Object obj) {
        if(this==obj) return true;
        if(!(obj instanceof Persona)) return false;
        Persona p = (Persona) obj;
        return nombre.equals(p.nombre) && edad==p.edad;
    }

    @Override
    public int hashCode() {
        return nombre.hashCode() + edad;
    }

    @Override
    public String toString() {
        return nombre + ", " + edad + " años";
    }
}

// Uso
Persona p1 = new Persona();
Persona p2 = new Persona();
System.out.println(p1.equals(p2));
System.out.println(p1.hashCode());
System.out.println(p1);
  • final: protege clases, métodos o variables.
  • equals(): compara objetos por contenido.
  • hashCode(): asegura que objetos sean manejables en colecciones.
  • toString(): define la representación textual del objeto.

11. Paquetes y Librerías

Los paquetes organizan clases en directorios lógicos y las librerías permiten reutilizar código existente.


// Declaración de paquete
package misclases.utilidades;

// Importar librerías
import java.util.ArrayList;
import java.util.Scanner;

// Uso
ArrayList lista = new ArrayList<>();
Scanner sc = new Scanner(System.in);

Paquetes y librerías permiten mantener código organizado y reutilizable, evitando duplicación y errores.
